Auth

Event
Action
Data

Login

Sent to authenticate user

Direction Client
Message A|REQ|loginData+

Login Success

Recieved when login passes

Direction Server
Message A|A+

Invalid Login

Recieved if login fails

Direction Server
Message A|E|INVALID_AUTH_DATA|unknown user+

Too many login attempts

Recieved if login fails more than the maximum allowed attempts and closes the client connection

Direction Server
Message A|E|TOO_MANY_AUTH_ATTEMPTS+

Events

Event
Action
Data

Listen

Send when listening to event matching regex

Direction Client
Message E|L|event\/.*+
Ack E|A|L|event\/.*+

Unlisten

Send when no longer interested in listening to event

Direction Client
Message E|UL|event\/.*+
Ack E|A|UL|event\/.*+

Susbcribe

Send to subscribe to event

Direction Client
Message E|S|eventName+
Ack E|A|S|eventName+

Unsubscribe

Send to unsubscribe to event

Direction Client
Message E|US|eventName+
Ack E|A|US|eventName+

Publish

Recieved/Sent when an event occurs

Direction Client/Server
Message E|EVT|eventName|JSON+
Ack E|A|EVT|eventName+

Listen Match

Recieved when a listened to regex is has a single intrested client

Direction Server
Message E|SP|event\/.*|event/A+

Listen UnMatched

Recieved when a listened to regex no longer has any interested clients

Direction
Message E|SR|event\/.*|event/A+

Record

Event
Action
Data

Listen

Direction Client
Message R|L|recordPrefix\/.*+
Ack R|A|L|recordPrefix\/.*+

Unlisten

Direction Client
Message R|UL|recordPrefix\/.*+
Ack R|A|UL|recordPrefix\/.*+

Listen Match

Recieved when a listened to regex is matched

Direction Server
Message R|SP|recordPrefix\/.*|/A+

Listen UnMatched

Recieved when a listened to regex no longer has any interested clients

Direction Server
Message R|SR|RecordPrefix\/.*|recordPrefix/recordName+

Read/Create

Send to request the record data, automatically subscribing to all changes and creates record if it does not exist

Direction Client
Message R|CR|recordName+
Ack R|A|CR|recordName+

Update Entire Record

Send/Recieved when the entire record data is replaced with a new image

Direction Client
Message R|U|recordName|versionNumber|JSON+

Patch Record

Send/Recieved when the record is partially changed

Direction Client
Message R|P|recordName|versionNumber|path|value+

Delete Record

Send/Recieved when an object is deleted. This leads to all clients discarding the client since updates are no longer available

Direction Client
Message R|D|recordName+
Ack R|A|D|recordName+

Receive Record Read

When requesting a record you will get a read which is the current state of the record, similar to the update data

Direction Server
Message R|R|recordName|versionNumber|JSON

Cache Read Timeout

If the object can't be read from the cache within a certain time the server will respond with this error

Direction Server
Message R|E|CACHE_RETRIEVAL_TIMEOUT|recordName+

Database Read Timeout

If the object can't be read from storage within a certain time the server will respond with this error

Direction Server
Message R|E|STORAGE_RETRIEVAL_TIMEOUT|recordName+

RPC

Event
Action
Data

Register as provider

Send when client can recieve RPC events

Direction Client
Message P|S|rpcName+
Ack P|A|S|rpcName+

Unregister as provider

Send when client no longer wishes to recieve RPC events

Direction Client
Message P|US|rpcName+
Ack P|A|US|rpcName+

Send RPC request

Send/Recieve when RPC requested

Direction Client/Server
Message P|REQ|rpcName|rpcID|Arguments+
Ack P|A|REQ|rpcName|rpcID+

Reject RPC request

Send if client does not want to execute RPC

Direction Client
Message P|REJ|rpcName|rpcID+

Error RPC request

Send/Recieved if an error occurs while attempting to execute rpc

Direction Client/Server
Message P|E|ErrorMessage|rpcName|123+

Respond to RPC request

Send/Recieve when RPC is executed succesfully with the resulting data

Direction Client/Server
Message P|RES|rpcName|rpcID|Result+
